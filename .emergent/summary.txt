The user wants to build a comprehensive Accounting, Inventory, and Point of Sale (POS) web application for a multi-branch business.

**PRODUCT REQUIREMENTS:**
- **Offline POS:** Must function offline and sync data automatically.

- **Multi-Branch Management:** Support multiple business locations with branch-specific data and an owner-level overview.
- **Advanced Accounting:** Track expenses, receivables, and manage funds. Includes workflows for farm expenses and customer cash advances that auto-generate invoices.

- **Daily Operations:** Sequential sales log, daily profit report, and a Close Accounts feature (Z-Report).
- **Granular Permissions:** An Inflow-Cloud-style system to control user access to specific features.
- **Editable Invoices & POs:** Ability to edit posted invoices (with an audit trail) and a Reopen PO feature to reverse inventory, allow edits, and re-receive stock.
- **Inventory Count Sheets:** A feature to perform physical inventory counts and record variances.
- **Employee & User Management:** Create and manage user accounts with PINs. Track employee cash advances with manager approval workflows.
- **Supplier Payments:** A dedicated section to pay suppliers for Purchase Orders.
- **Data Import:** Ability to import data (like products) from external sources like QuickBooks via Excel/CSV.
