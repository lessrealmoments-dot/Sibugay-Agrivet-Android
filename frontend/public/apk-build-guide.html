AgriSmart Terminal — APK Build Guide

Step-by-step instructions to build and install the H10P Android terminal app

📱 Target Device: H10P Android 14 POS 🌐 App URL: agri-books.com/terminal 📦 Output: app-debug.apk

⚠️ Before You Start — Things You Need Ready

💻 Windows 11 computer
🌐 Stable internet connection
📄 printer-release.aar file (from Senraise supplier)
⏱️ About 1–2 hours (mostly waiting for downloads)
🔗 GitHub repo: github.com/lessrealmoments-dot/Sibugay-Agrivet-supply-tenant
📱 H10P device + USB cable

PHASE 1 Install Required Tools

1
Install Git

If Git is not already installed, download it from:

https://git-scm.com/download/win

Run the installer → click Next through all screens → Install → Finish

Then restart your computer after installing.

2
Install Node.js

Go to: https://nodejs.org

Click the big green LTS button → download the .msi file → run it.

Click: Next → Next → Next → Install → Finish

Verify it worked: Open CMD (press Windows key, type cmd, press Enter) and type:

node --version

You should see something like v20.18.0 ✅

3
Install Java 17 (Required by Android Studio)

Go to: https://adoptium.net/temurin/releases/?version=17

Select: Version = 17 LTS · OS = Windows · Architecture = x64 · Package = JDK

Download the .msi file → run it → Next → Next → Install → Finish

⚠️ Important: Do NOT install Java 11 or Java 21. Must be exactly Java 17.
4
Install Android Studio

Go to: https://developer.android.com/studio

Download the latest version (should say Android Studio Ladybug or newer — 2024.x or later).

Run the installer → Next → Next → Install → Finish → let it launch.

In the setup wizard:

  • Select "Do not import settings"
  • Choose "Standard" installation
  • Click Next → Finish
  • Wait for the Android SDK to download (2–4 GB — get a coffee ☕)
⚠️ Version matters: Must be Android Studio Ladybug (2024.2.x) or newer. Older versions will fail.

PHASE 2 Download the Code

5
Open Command Prompt (CMD)

Press the Windows key on your keyboard → type cmd → press Enter

A black window opens. This is your Command Prompt.

6
Go to Documents Folder

Type this exactly and press Enter:

cd "C:\Users\Sibugay Agrivet\Documents"

The line now shows: C:\Users\Sibugay Agrivet\Documents>

7
Clone the Repository from GitHub

Type this exactly and press Enter:

git clone https://github.com/lessrealmoments-dot/Sibugay-Agrivet-supply-tenant "SIBUGAY AGRICULTURAL SUPPLY ANDROID APP"

You will see lines like "Receiving objects: 100%" — wait for it to finish.

8
Go Into the Project

Type these two commands one by one, pressing Enter after each:

cd "SIBUGAY AGRICULTURAL SUPPLY ANDROID APP"
cd frontend

Your CMD now shows:

C:\Users\Sibugay Agrivet\Documents\SIBUGAY AGRICULTURAL SUPPLY ANDROID APP\frontend>

PHASE 3 Set Up the Project

9
Install Project Dependencies

Still in CMD (you should be in the frontend folder), type:

npm install --legacy-peer-deps

Wait for it to finish. You'll see a lot of text — that's normal. Wait until the > prompt comes back.

ℹ️ The --legacy-peer-deps flag is needed to skip a minor version conflict. It's safe.
10
Copy the Printer SDK File

Open File Explorer and navigate to:

C:\Users\Sibugay Agrivet\Documents\SIBUGAY AGRICULTURAL SUPPLY ANDROID APP\frontend\android\app\libs

You will see a file called README.txt inside. Copy your printer-release.aar file (from your Senraise SDK folder) into this libs folder.

After this, the libs folder should contain:

📁 libs
   📄 README.txt
   📄 printer-release.aar   ← you just added this
⚠️ Critical: Without this file, the build will fail with an error about missing printer-release.
11
Create the App Config File

In File Explorer, navigate to:

C:\Users\Sibugay Agrivet\Documents\SIBUGAY AGRICULTURAL SUPPLY ANDROID APP\frontend\android\app\src\main

Check if a folder called assets exists. If not, right-click → New → Folder and name it assets.

Open the assets folder. Right-click → New → Text Document. Name it:

capacitor.config.json

⚠️ Make sure it ends in .json not .json.txt. Enable file extensions via View → File name extensions in File Explorer to check.

Right-click the file → Open with → Notepad. Delete everything and paste this:

{
  "appId": "com.agribooks.terminal",
  "appName": "AgriSmart Terminal",
  "webDir": "build",
  "server": {
    "url": "https://agri-books.com/terminal",
    "cleartext": false,
    "androidScheme": "https"
  },
  "android": {
    "captureInput": true,
    "webContentsDebuggingEnabled": false,
    "backgroundColor": "#1A4D2E"
  }
}

Press Ctrl + S to save. Close Notepad.

PHASE 4 Build the APK in Android Studio

12
Open the Android Project

Open Android Studio → click Open (NOT "New Project")

Navigate to and select the android folder:

C:\Users\Sibugay Agrivet\Documents\SIBUGAY AGRICULTURAL SUPPLY ANDROID APP\frontend\android

Click OK.

If a pop-up asks "Trust and open Gradle project?" → click Trust Project

13
Set Java 17 as the Gradle JDK

In Android Studio: File → Project Structure → SDK Location

Find the Gradle JDK dropdown. Select the option that says jbr-17, temurin-17, or any option with 17 in the name.

Click OK.

⚠️ Do NOT select "Embedded JDK" if it shows version 11. That will cause a build error.
14
Wait for Gradle Sync

At the bottom of Android Studio, a progress bar says "Gradle sync in progress..."

Wait for it to disappear completely — first time takes 5–10 minutes (downloads build tools).

When done, the bottom status bar shows no errors.

15
Build the APK

In the menu bar: Build → Build Bundle(s) / APK(s) → Build APK(s)

Wait 1–3 minutes. A progress bar appears at the bottom.

✅ When done, a notification appears bottom-right: "Build successful"
Click the word "locate" — it opens the folder with your APK file.

Your APK is at:

frontend\android\app\build\outputs\apk\debug\app-debug.apk

PHASE 5 Install on H10P Device

16
Allow Unknown Sources on H10P

On your H10P device:

  1. Open Settings
  2. Go to Security (or Privacy)
  3. Find and enable "Install Unknown Apps" or "Unknown Sources"

This allows installing apps from outside the Google Play Store.

17
Copy APK to H10P
  1. Connect H10P to your computer via USB cable
  2. On H10P screen: drag down notifications → tap "USB" → select "File Transfer"
  3. On your computer: open File Explorer → find the H10P device on the left side
  4. Open it → go to Internal Storage → Download
  5. Copy app-debug.apk into that folder
18
Install the APK
  1. On H10P: open File Manager (or Files app)
  2. Go to Downloads
  3. Tap app-debug.apk
  4. Tap Install
  5. Tap Open when finished

PHASE 6 First Launch & Pairing

19
Pair the Terminal to Your Branch
  1. Open AgriSmart Terminal on the H10P — it shows a screen with a 6-character code
  2. On your PC browser, go to https://agri-books.com and log in
  3. Go to Settings → Connect Terminal
  4. Type in the 6-character code → select your branch → confirm
  5. The H10P switches to the Sales tab — you're live! ✅
✅ You now have: Sales with barcode scanner · PO Check · Branch Transfers · QR Scanner with instant reprint

ERRORS Common Problems & Fixes

Error Message Fix
npm error ERESOLVE unable to resolve dependency tree Run npm install --legacy-peer-deps instead of npm install
class file version 61.0, only recognizes up to 55.0 Install Java 17 from adoptium.net → set Gradle JDK to temurin-17 in Project Structure
Incompatible AGP version, latest supported is AGP 7.2.1 Update Android Studio to latest version (Ladybug 2024.x or newer)
capacitor.settings.gradle does not exist Pull latest code: git pull then retry Gradle sync
Failed to resolve: project :capacitor-cordova-android-plugins Pull latest code: git pull — fixed version is in the repo
Could not resolve printer-release (aar) Copy printer-release.aar from Senraise SDK into android/app/libs/
Webpage not available — localhost Create capacitor.config.json manually (see Step 11) then Clean Build + Rebuild
App opens to full website instead of terminal Make sure URL in capacitor.config.json ends with /terminal
Build fails after any change Build → Clean Project → then Build → Build APK(s)
Printer says "service not connected" Ask Senraise supplier for recieptservice.apk and install it on H10P first

UPDATES How to Update the App in the Future

Web app updates (new features, bug fixes): Just deploy to agri-books.com — the APK picks them up automatically on next launch. No APK rebuild needed.


Printer plugin updates: Rebuild the APK and reinstall on H10P.

To deploy web app updates to Hostinger, SSH in and run:

cd /var/www/agribooks && git pull origin main
cd frontend && yarn install && yarn build
cd /var/www/agribooks && supervisorctl restart agribooks-backend
